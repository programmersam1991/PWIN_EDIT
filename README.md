# Домашнее задание к занятию "Настройка виртуальных машин"

### Цель задания

Инструмент для виртуализации Oracle VM VirtualBox даёт широкие возможности для разворачивания систем автоматизации. С помощью этого инструмента можно развернуть полноценную SCADA-систему на персональном ПК и произвести необходимые настройки на виртуальном аналоге до полноценного внедрения на производство. Это позволит избежать многих ошибок и проблем по совместимости ПО.

В результате выполнения этого задания вы сможете:
1.	Развернуть собственную виртуальную машину на базе ОС Windows.
2.	Настроить связь виртуальной машины с рабочим компьютером.
3.	Создать снимок виртуальной машины.

### Задание 1. Импорт виртуальной машины

1.	Изучите материалы занятия.
2.	Скачайте [файл с экспортированной Windows 10](https://disk.yandex.ru/d/tWPIAYvFbIw9wg) (формат .ova) и импортируйте его на виртуальную машину. Данную виртуальную машину назовите "Win_10_OVA"
3.	Настройте функцию общего буфера обмена и переноса файлов в настройках виртуальной машины. Проверьте обмен файлами посредством функции drag&drop.
4.	Создайте на хостовом компьютере папку "VM_Shared" в любом удобном месте и привяжите данную папку к виртуальной машине, как общую.
5.	Добавьте в папку на хостовом компьютере файл с произвольным содержимым. Откройте данный файл на виртуальной машине из общей папки.

В качестве результата выполнения задания 1 необходимо будет предоставить:

- снимок экрана запущенной виртуальной машины, импортированной из файла .ova.
- снимки экрана настроек общей папки и функции drag&drop.
- снимок содержимого общей папке на хостовом компьютере и на виртуальной машине.

### Задание 2. Установка операционной системы на виртуальную машину

1.	Изучите материалы занятия.
2.	Создайте чистую виртуальную машину с именем "Win_10_ISO" и выполните необходимые настройки.
3.	Скачайте [файл образа Windows 10](https://disk.yandex.ru/d/tWPIAYvFbIw9wg) (формат .iso) и произведите его установку на виртуальную машину.

В качестве результата выполнения задания 2 необходимо будет предоставить:

- снимок экрана запущенной виртуальной машины, установленной из образа (формат .iso)
- настройки виртуальной машины "Win_10_ISO" (графа "Носители").

### Задание 3. Создание snapshot`а виртуальной машины

1. Для этого задания используйте виртуальную машину "Win_10_OVA". 
2. Создайте снимок (snapshot) чистой виртуальной машины "Win_10_OVA". Данный снимок назовите "Snap_Empty"
3. Загрузите с помощью средств обмена данными (общей папки или переноса файлов) произвольные текстовые файлы на рабочий стол.
4. Создайте снимок (snapshot) текущего состояния виртуальной машины. Данный снимок назовите "Snap_Fill"
5. Выполните откат к снимку "Snap_Empty".
6. Вернитесь к снимку "Snap_Fill"

В качестве результата выполнения задания 3 необходимо будет предоставить:
- снимок экрана созданных снимков VirtualBox.
- снимок экрана рабочего стола виртуальной машины "Win_10_OVA" с активным снимком "Snap_Empty".
- снимок экрана рабочего стола виртуальной машины "Win_10_OVA" с активным снимком "Snap_Fill".

### *Задание 4. Настройка сетевых адаптеров виртуальных машин

1. Изучите материалы занятия.
2. Выполните настройку сетевого адаптера виртуальной машины "Win_10_OVA" для связи с хостом.
3. Проверьте связь "Win_10_OVA" с хостовым компьютером. Для этого выполните команду _ping <host_ip_address>_ из командной строки виртуальной машины "Win_10_OVA".
4. Проверьте связь хостового компьютера с виртуальной машиной "Win_10_OVA". Для этого выполните команду _ping <Win_10_OVA_ip_address>_ из командной строки хостового компьютера.
5. Выполните настройку сетевого адаптера виртуальной машины "Win_10_ISO" для связи с виртуальной машиной "Win_10_OVA".
6. Проверьте связь виртуальной машины "Win_10_ISO" с виртуальной машиной "Win_10_OVA". Для этого выполните команду _ping <Win_10_OVA_ip_address>_ из командной строки виртуальной машины "Win_10_ISO".
7. Проверьте связь виртуальной машины "Win_10_OVA" с виртуальной машиной "Win_10_ISO". Для этого выполните команду _ping <Win_10_ISO_ip_address>_ из командной строки виртуальной машины "Win_10_OVA".

В качестве результата выполнения задания 4 необходимо будет предоставить:

- снимок экрана CLI после введенной команды _ping <host_ip_address>_ из виртуальной машины "Win_10_OVA".
- снимок экрана CLI после введенной команды _ping <Win_10_OVA_ip_address>_ из хоста.
- снимок экрана CLI после введенной команды _ping <Win_10_OVA_ip_address>_ из виртуальной машины "Win_10_ISO".
- снимок экрана CLI после введенной команды _ping <Win_10_ISO_ip_address>_ из виртуальной машины "Win_10_OVA".
